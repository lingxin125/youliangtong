农户收款账户管理 PRD

将农户端「结算信息修改」与「数币账户维护」合并为统一的「收款账户管理」，页面分「银行账户」与「数币账户」两个分区。农户建档后银行账户不再作为必填前置：可为空；为空时进入独立新增页维护一张唯一银行账户，新增后不可删除、不可新增第二张，仅支持沿用现有模式逐项修改；数币账户最多一个：为空时可新增，已有后仅支持删除重建，不支持编辑。V1.4 补充：银行账户新增按农户商户类型控制可选账户类型，个人商户仅个人银行卡，企业 / 对公商户可选个人银行卡或对公账户。

1. 背景与目标

现状问题入口分散

农户端「我的」页同时存在「账户管理」和「结算信息修改」两个入口，账户能力分散在两个页面，口径不一致：账户管理页允许多卡和删除，与农户实际「建档即一卡」的业务不符。

合并目标单一入口

合并为唯一入口「收款账户管理」，内含银行结算账户（唯一、改不删）与数币收款账户（最多一个、增删不改），口径与农户建档业务对齐。

设计原则：尽量不改动现有交互模式。银行卡修改完全沿用当前结算信息修改的「逐项修改 + 底部弹层」模式；数币账户新增参考收购商付款账户管理的钱包新增逻辑，仅按农户场景补充字段。

3. 银行账户（唯一结算卡，改不删）

业务规则

  • 农户建档后银行账户允许为空；为空时展示空态卡片，并提供一次新增入口。
  • 一旦新增成功，银行账户即变为唯一结算卡：不可删除、不可新增第二张。
  • 空态视觉使用中性灰色虚线卡片 + 加号入口，不使用绿色强调，不展示解释性长文案。
  • 银行账户新增使用独立页面 farmer-bank-add.html，不在收款账户管理页内弹层新增。
  • 新增账户类型受农户商户类型限制：个人商户只能新增个人银行卡；企业 / 对公商户可新增个人银行卡或对公账户。
  • 新增页不直接展示「商户类型」字段，只展示「账户类型」区域；商户类型作为后台规则决定可选账户类型。
  • 个人银行卡需区分「本人银行卡 / 非本人银行卡」，用于后续合规校验和资料留存；对公账户不展示本人 / 非本人归属。
  • 卡片头部布局与收购商账户管理的银行卡卡片一致：银行图标 + 银行名称 + 个人户 / 对公户标签，不带副标题说明文案。
  • 修改沿用现有结算信息修改模式：每次仅修改一项，底部弹层内展示当前值并填写新值。
  • 修改弹层中的「当前值」默认脱敏展示，右侧提供小眼睛图标，点击切换明文 / 脱敏后再进行修改。

新增银行账户规则（farmer-bank-add.html）

农户商户类型可选账户类型页面展示字段口径
个人商户个人银行卡不展示商户类型；账户类型区域仅展示一个选中的大卡片「个人账户」；展示本人 / 非本人银行卡归属和提示文案姓名、手机号、身份证号、银行卡号、开户行；小行需开户支行
企业 / 对公商户个人银行卡、对公账户不展示商户类型；账户类型区域展示「个人账户 / 对公账户」两个卡片；选择个人账户时继续展示本人 / 非本人归属个人银行卡同上；对公账户填写对公账户名、银行账号、开户行；小行需开户支行

开户行 / 开户支行展示逻辑（V1.1 明确）

  • 开户行一定展示；开户支行是否展示取决于开户行能否通过银行卡号自动识别。
  • 大行（邮储 / 工行 / 农行 / 中行 / 建行等）：卡号可自动带出开户行 → 只展示开户行，不展示开户支行。
  • 小行（卡号无法自动识别）：需手动选择开户行和开户支行 → 开户行、开户支行均展示。

修改银行卡号的弹层交互（参考农户建档）

1「开户行」字段默认常驻展示、初始为空；开户支行不展示
2输入卡号自动识别成功 → 开户行自动回显，仍不展示开户支行
3识别不出 → 开户行切换为手动下拉，并展示开户支行下拉；不附加任何提示文案

4. 数币账户（最多一个，增删不改）

业务规则

  • 数币账户最多持有一个：已有数币账户时隐藏新增按钮，需先删除再新增。
  • 数币账户不支持编辑：信息变更通过「删除 → 重新新增」完成。
  • 删除需底部弹层二次确认。
  • 新增逻辑参考收购商付款账户管理的钱包新增（含钱包 ID 校验），并按农户场景扩展字段。

空态与新增入口（V1.1 调整）

  • 无数币账户时，数币账户区展示空态卡片，并在卡片内部提供加号新增入口。
  • 不再使用页面底部固定新增按钮；新增动作收口在空态卡片内部。
  • 已有数币账户时空态入口消失（最多一个，无可新增项）。

钱包卡片回显规则（V1.1 明确）

  • 卡片标题固定为「数字人民币钱包」+ 账户类型标签（个人 / 对公），与收购商端钱包卡片保持一致。
  • 副标题行：所属银行 · 钱包等级（如「中国邮政储蓄银行 · III类钱包」）。
  • 详情区第一行为钱包名称（明文展示）——农户作为收款人时钱包名称是必填必传项，必须直观可见；其后依次为钱包 ID、姓名、手机号、身份证号（均脱敏）。

6. 数据与 Mock 口径

  • 数币钱包存储键：ylt_farmer_wallet_v2（单对象，跨页共享给新增页；删除即移除该键）。银行卡本地演示键：ylt_farmer_bank_account_v1。
  • 种子数据调整为：默认银行账户为空、数币账户为空，便于演示两个空态新增流程。
  • 卡号自动识别 Mock（BIN 前缀）：6215 / 6221 / 6228 → 邮储，6222 → 工行，6230 → 农行，6216 → 中行，6217 → 建行；其余前缀视为无法识别，进入手动选择开户行 + 开户支行。
  • 演示用钱包 ID：008188889012 / 008188887777（通过）、008188885678（非实名提示）、其他（未找到钱包提示）。
  • 当前原型保留「演示公户 / 演示个人户」和「演示银行有值 / 空态」悬浮切换，便于评审时快速查看不同账户形态；新增页仍按真实商户类型规则控制可选账户类型。
  • 验证码 Mock 固定为 123456（如涉及）。

7. 验收标准

  • 农户「我的」页只保留一个「收款账户管理」入口，原「账户管理」「结算信息修改」两个入口移除。
  • 页面分区标题为「银行账户」「数币账户」；银行卡卡片头部为银行图标 + 银行名称 + 账户类型标签，无副标题说明文案。
  • 银行账户为空时展示灰色虚线「添加银行账户」空态卡片；点击进入新增银行账户独立页，新增成功后切换为唯一银行卡卡片，且不再出现删除 / 新增能力。
  • 个人商户进入新增银行账户页时，只能看到一个选中的「个人账户」大卡片，不展示商户类型和对公账户入口；企业 / 对公商户进入时，可选择个人账户或对公账户。
  • 修改弹层当前值默认脱敏，小眼睛可切换明文；新值输入框为空，不回填旧值。
  • 修改银行卡号：「开户行」字段常驻（初始为空），输入大行卡号自动回显开户行且不出现开户支行；输入无法识别的卡号时开户行变为手动下拉并出现开户支行下拉，无附加提示文案；保存后卡片按对应规则展示 / 隐藏开户支行行。
  • 无数币账户时展示卡片内加号新增入口；新增校验通过后返回列表并展示钱包卡片。
  • 钱包卡片标题为「数字人民币钱包」+ 类型标签，详情第一行明文展示钱包名称。
  • 数币账户删除需二次确认；删除后恢复空态卡片内加号新增入口。
  • 钱包 ID 三种校验分支提示文案正确。

8. 待确认问题

  • 对公数币账户的「姓名」是否需要改为企业 / 账户名称？当前方案两种类型字段一致。
  • 删除数币账户是否需要追加手机号验证码校验？当前方案仅二次确认弹窗。
  • 数币账户是否参与收款码 / 收款场景的账户选择？当前方案仅做维护，不引入默认账户切换概念。
