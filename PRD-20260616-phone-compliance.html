Compliance PRD

丰收云小程序手机号登录合规整改

本需求用于响应微信小程序手机号收集合规预警。最终采用最小整改口径：保留原登录入口结构和双按钮样式，仅在入口页显性展示协议勾选，并新增一行内部商户使用范围说明。

需求编号REQ-088
模块登录
目标日期2026-06-16
状态需求设计

1. 背景与依据

微信预警指出小程序可能存在违规收集手机号的问题，常见风险包括进入小程序即要求手机号、未说明必要原因、拒绝后无法退出流程等。丰收云实际为邮政机构客户经理协助进件后的签约商户服务工具，业务功能不面向公众开放浏览。

  • 官方规范：关于小程序收集用户手机号行为的规范
  • 运营规范：微信小程序平台运营规范
  • 接口说明：手机号快速验证组件

2. 目标与范围

目标

  • 保持原登录入口的视觉结构，不新增大段合规说明卡片。
  • 在登录入口显性展示用户协议与隐私政策勾选项。
  • 在协议下方新增内部商户说明：本小程序为内部商户小程序，需加入商户团队后使用。
  • 登录入口两个按钮均需先勾选协议后才能继续。
  • 支撑审核识别丰收云为内部商户团队使用场景，同时降低现有用户的认知变化。

不包含

  • 不新增公开游客模式。
  • 不开放普通用户自助注册。
  • 不展示客户经理个人联系方式。
  • 不新增“签约商户登录”卡片、登录说明卡片或长篇提示文案。
  • 不改动登录后的开单、订单、账户等业务主链路。

3. 页面文案

位置文案说明
平台副标题删除“农产品收购交易平台”减少页面信息密度，回到最小整改。
手机号登录按钮手机号快捷登录保留原入口按钮样式和名称。
一键登录按钮一键登录保留现有名称。
协议勾选我已阅读并同意《用户协议》和《隐私政策》展示在两个登录按钮下方。
内部商户说明本小程序为内部商户小程序，需加入商户团队后使用灰色小字，左对齐，放在协议下方。

4. 业务规则

  • 进入小程序和登录入口页时，不自动弹出手机号授权。
  • 未勾选协议时，点击 手机号快捷登录 不进入手机号登录页，并提示先同意协议。
  • 未勾选协议时，点击 一键登录 不触发手机号授权，并提示先同意协议。
  • 勾选协议后，手机号快捷登录 进入现有手机号登录页；一键登录 触发现有微信手机号授权流程。
  • 继续沿用现有商户团队账号准入逻辑，未加入商户团队的账号不得进入业务功能。
  • 用户拒绝授权后，不反复拉起授权弹窗，可返回或改用手机号验证码登录。
  • 隐私政策明确手机号用途：签约账号核验、登录验证、交易安全校验和必要服务联系。

5. 提审材料

  • 服务内容录屏：登录入口 -> 勾选协议 -> 登录成功 -> 进入首页/订单/开单等业务页面。
  • 页面截图：登录入口、协议勾选、内部商户说明、隐私协议手机号说明。
  • 体验账号：提供已完成进件的测试手机号和固定验证码。
  • 审核说明：丰收云为内部商户团队使用的小程序，需先加入商户团队后使用，手机号用于登录验证、签约账号核验和交易安全校验。

6. 验收标准

  • 首次进入不自动请求手机号授权。
  • 登录入口保持原始 logo + 双按钮结构。
  • 不再展示“农产品收购交易平台”副标题。
  • 未勾选协议时，手机号快捷登录和一键登录均被拦截。
  • 协议下方可见内部商户小程序说明，且文案左对齐。
  • 未加入商户团队的账号无法进入业务功能。
  • 拒绝授权后不退出小程序，不重复弹窗。
  • 隐私协议展示手机号用途说明。
