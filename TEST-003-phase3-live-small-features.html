1. 测试准备

准备项要求
账号至少准备收购商超管、收购商业务员、农户三个账号。
渠道微信小程序、微信 H5 扫码、邮储银行 H5、邮储银行 App 各准备一轮首次登录和免登测试。
数据准备已建档农户二维码、未绑定有效收款账户农户、无效二维码。
订单准备挂单、待开价、待付款、付款中、已取消、付款成功订单。

2. 团队管理测试用例

ID优先级场景步骤预期结果
TM-001P0超管查看团队管理入口使用收购商超管登录，进入“我的”。展示“团队管理”入口，点击进入团队管理页。
TM-002P0业务员不可管理团队使用业务员账号登录，进入“我的”。不展示团队管理入口；直接访问团队管理地址时应拦截或提示无权限。
TM-003P1查看团队成员列表超管进入团队管理页。正确展示成员姓名、手机号、角色、状态；角色仅出现超管和业务员。
TM-004P1添加业务员点击添加业务员，录入有效手机号和成员信息并提交。添加成功后成员出现在列表中，角色为业务员。
TM-005P1当前团队重复添加录入已存在于当前团队的业务员手机号提交。系统提示成员已存在，不重复创建。
TM-006P0跨团队重复添加限制使用另一个收购商超管，添加已归属其他收购商团队的业务员手机号。系统拦截添加，并提示该手机号已加入其他团队或暂不支持重复加入。
TM-007P1删除业务员在团队列表中选择某个业务员并确认删除。删除成功后，该业务员从团队列表移除。
TM-008P1删除确认点击删除业务员后取消确认。取消后成员仍保留在列表中。
TM-009P1超管不可被普通删除查看超管成员操作项。超管成员不展示普通删除操作，或删除时明确拦截。

3. 结算信息修改测试用例

个人户

ID优先级场景步骤预期结果
SI-001P0入口展示分别进入收购商“我的”和农户“我的”。均展示“结算信息修改”入口，进入后返回来源正确。
SI-002P0敏感信息脱敏进入个人户结算信息修改页。姓名、手机号、身份证号、银行卡号默认脱敏；开户行、支行明文展示。
SI-003P0修改姓名点击姓名修改，输入当前手机号验证码，通过后输入新姓名确认。验证码正确才进入下一步；修改成功后页面回显新姓名。
SI-004P0修改手机号点击手机号修改，输入原手机号验证码、新手机号、新手机号验证码。两个验证码均通过后修改成功；任一错误则拦截。
SI-005P0修改身份证号点击身份证号修改，先校验当前手机号验证码，再录入新身份证号。校验通过后展示未脱敏原身份证号；新身份证号格式错误时拦截。
SI-006P0修改银行卡号点击银行卡号修改，先校验当前手机号验证码，再录入新卡号并选择开户行、支行。必须重新选择开户行和开户支行；成功后回显新卡号脱敏信息。
SI-007P1错误验证码任一字段修改时输入非 123456 验证码。提示验证码错误，不能进入下一步或提交成功。
SI-008P1单字段修改限制开始修改一个字段后尝试修改另一个字段。同一时间只允许一个字段进入修改流程。

对公户

ID优先级场景步骤预期结果
SI-101P1对公户展示切换到对公户演示形态。展示对公账户名称、手机号、银行账号、开户行、开户支行。
SI-102P0修改对公账户名称点击对公账户名称修改，输入当前手机号验证码和新名称。验证码正确后修改成功。
SI-103P0修改对公银行账号点击银行账号修改，校验手机号验证码后输入新账号并选择开户行、支行。账号、开户行、支行都填写后才可提交成功。
SI-104P1开户行支行不可单独修改查看开户行、开户支行区域。不展示独立修改按钮，也不展示“不可修改”文案。

4. 多端登录态 / 免登 / 互踢测试用例

测试口径：免登优先按“同一渠道容器 + 同一设备 + 有效登录态”验证。跨渠道是否互踢以技术最终实现为准，本组用例需要记录实际结果并与技术确认。
ID优先级场景步骤预期结果
LS-001P0微信 H5 首次扫码登录清除微信 H5 登录态，使用微信扫一扫进入扫码开单。首次进入需要登录；登录成功后进入开单页。
LS-002P0微信 H5 同设备免登保持微信 H5 登录态，再次使用同一微信扫描农户二维码。不再要求登录，直接进入开单页。
LS-003P0邮储银行 App 首次扫码登录清除邮储银行 App 内登录态，使用邮储银行 App 扫码。首次进入需要登录；登录成功后进入开单页。
LS-004P0邮储银行 App 同设备免登保持邮储银行 App 登录态，再次扫码。不再要求登录，直接进入开单页。
LS-005P1微信 H5 登录后打开微信小程序先完成微信 H5 扫码登录，再打开丰收云微信小程序。微信小程序不应直接复用微信 H5 登录态；如需登录，应独立完成小程序登录。
LS-006P1微信小程序登录后微信 H5 扫码先登录微信小程序，再用微信扫一扫进入扫码开单 H5。微信 H5 是否免登需按 OpenID / H5 登录态判断；不得因小程序已登录而错误跳过 H5 登录校验。
LS-007P1微信侧登录后邮储银行 App 扫码先完成微信小程序或微信 H5 登录，再使用邮储银行 App 扫码。邮储银行 App 应独立判断登录态；首次进入时需要重新登录。
LS-008P1邮储银行 App 登录后微信侧访问先完成邮储银行 App 扫码登录，再打开微信小程序或微信 H5。微信侧应独立判断登录态；不得直接复用邮储银行 App 登录态。
LS-009P1同账号多端互踢验证同一业务员账号依次登录微信小程序、微信 H5、邮储银行 H5、邮储银行 App，并回到前一个端继续操作。记录各端是否被退出；若无 OpenID / 渠道 ID 的端发生互踢，需要与技术确认属于预期。
LS-010P1OpenID / 渠道 ID 绑定免登验证使用已完成授权绑定的同一渠道再次进入扫码开单。系统能基于该渠道身份识别用户并免登；换渠道时不应误用其他渠道身份。

5. 扫码开单测试用例

ID优先级场景步骤预期结果
SB-001P0微信首次扫码清除登录态，使用微信扫描有效农户二维码。先进入登录流程；登录成功后自动进入开单页。
SB-002P0微信再次扫码免登保持登录态，再次用微信扫描有效农户二维码。不再要求登录，直接进入开单页。
SB-003P0邮储银行 App 首次扫码清除登录态，使用邮储银行 App 扫描有效农户二维码。先进入登录流程；登录成功后自动进入开单页。
SB-004P0邮储银行 App 再次扫码免登保持登录态，再次用邮储银行 App 扫描有效农户二维码。直接进入开单页。
SB-005P0扫码后页面限制扫码进入后观察页面和可操作按钮。只进入开单页；仅允许暂存或直接付款，不跳到订单列表或其他中间页。
SB-006P0农户信息自动带入扫描已建档农户二维码进入开单页。收款人信息自动回填，默认收款账户可用于付款。
SB-007P1暂存开单扫码进入开单页，填写必要信息后点击暂存。订单进入挂单 / 待处理链路，可在订单管理中查看。
SB-008P0直接付款扫码进入开单页，填写完整信息后点击直接付款。进入确认付款 / 收银台链路，付款结果正常回写订单状态。
SB-009P1无效二维码扫描无效、过期或无法解析二维码。提示二维码无效，不进入开单页。
SB-010P1未加入团队手机号首次扫码清除登录态，使用未加入任何收购商团队的手机号扫码后尝试登录。登录阶段无法完成账号识别或登录，不进入开单页。

6. 订单管理关联测试用例

ID优先级场景步骤预期结果
OM-001P0页签结构进入收购商订单管理。一级页签为待处理、付款中、已结束。
OM-002P1待处理编辑分别查看挂单、待开价、待付款订单。三类订单均支持编辑信息。
OM-003P0付款中批量付款进入付款中，勾选多笔订单，点击批量付款。支持多选、全选；跳转确认付款页并携带所选订单。
OM-004P1部分失败后重试模拟批量付款部分失败，返回订单管理。失败订单仍停留付款中，可再次勾选并批量付款。
OM-005P1已取消展示查看已结束中的取消订单。统一展示为已取消，不区分付款前取消 / 付款中取消。
OM-006P2按钮文案查看待付款和付款中底部操作栏。付款按钮统一展示为“批量付款”，不出现“合并付款”。

7. 回归检查

模块检查点预期
登录扫码登录后返回目标页。首次登录后能回到开单页，不丢失二维码带入的农户信息；不同渠道登录态按各自容器独立验证。
开单普通入口开单。不受扫码开单影响，仍可手动选择农户、暂存、付款。
支付确认付款、收银台、付款结果。直接付款链路正常，状态回写正确。
权限超管和业务员菜单。不同角色菜单和操作权限正确。
账户账户管理与结算信息修改。两个入口互不影响，修改结算信息不破坏账户列表展示。
建议测试优先级：先跑 SB-001 至 SB-008，再跑 SI-001 至 SI-006，最后跑团队管理和订单管理回归。
