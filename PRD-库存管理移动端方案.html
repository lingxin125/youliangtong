库存管理移动端方案

移动端不复制 PC 库存维护能力，只补齐开单选择入库仓库、订单详情记录入库仓库，以及“我的 > 库存管理”下的库存概览、库存台账、库存明细、农户存粮只读查询。

入口位置
我的 > 库存管理
操作边界
只读查询 + 开单选择
库存状态建议
预入库 / 待确认入库
订单详情
仅展示入库仓库

1. 设计结论

入口不放首页低频管理

库存入口收口到我的模块，避免轻量收购用户过早感知复杂仓储能力。

首页轻预览最近 5 条

库存首页只放概览和三 Tab 预览；搜索、筛选、汇总全部下沉到列表页。

库存明细强调语义入库出库

右上角用入库 / 出库表达库存方向，左下角用收粮 / 存粮表达库存来源。

复杂操作仍在 PC只读查询

库存调整、出库、调拨、盘点、农户存粮新增和存转销继续由 PC 端完成。

2. 范围与权限

范围本期处理说明
开单页新增入库仓库字段可为空，不选择则不生成预入库；强库存客户后续可配置必填。
订单详情有仓库才展示仅在订单信息中展示入库仓库，不拆库存处理模块。
我的页新增库存管理入口仅超管和有库存查询权限账号可见。
库存查询概览、台账、明细、农户存粮只读，不做新增、编辑、删除、调拨、盘点、出库、存转销。
角色入口可见性数据范围
超管可见当前收购商主体下全部有权限仓库。
有库存查询权限账号可见按账号授权仓库范围查看。
无库存查询权限账号不展示入口后端接口仍需拒绝直链访问。
前端入口隐藏只是体验控制，后端接口必须按账号权限和可见仓库范围过滤数据。

3. 开单入库流程

字段规则
入库仓库下拉选择，仅展示启用仓库；可为空，不选择则不生成预入库。
仓库来源PC 端仓库接口，仅展示启用仓库。
历史禁用仓库新开单不展示；历史订单按订单快照展示仓库名称。
提交字段选择仓库后提交 `warehouseId` 和 `warehouseName`。
flowchart TD
    A[收购商进入开单页] --> B{商户是否启用入库仓库配置}
    B -- 否 --> C[不展示入库仓库字段]
    C --> D[正常开单/挂单/待开价/付款]
    B -- 是 --> E[展示入库仓库字段]
    E --> F{是否选择仓库}
    F -- 否 --> G[订单不记录入库仓库]
    G --> H[订单详情不展示入库仓库]
    F -- 是 --> I[提交 warehouseId 和 warehouseName]
    I --> J[订单记录入库仓库快照]
    J --> K[订单详情订单信息展示入库仓库]
    K --> L[后端生成预入库或待确认入库数据]
    L --> M[PC端完成正式库存确认或后续库存作业]
    M --> N[移动端库存管理只读查询]
底层状态建议含义移动端展示
未入库开单未选择仓库，订单不进入库存流程。订单详情不展示入库仓库。
预入库 / 待确认入库已记录仓库归属，等待后端或 PC 正式确认。库存概览可计入待确认入库。
已入库已正式入库，形成库存流水。在库存台账 / 库存明细中查询。
入库取消 / 冲销后续库存处理被撤销或冲销。在库存明细或 PC 端查看，不在订单详情展开。

4. 订单详情展示边界

订单详情只记录本单农产品入到哪个仓库，不展示库存处理过程。

场景订单详情展示
开单未选择仓库不展示 `入库仓库` 行。
开单选择仓库在 `订单信息` 区域展示 `入库仓库`。
仓库后续被禁用仍展示订单保存时的仓库名称快照。
后续库存确认、失败、冲销不在订单详情展示，进入库存管理或 PC 库存功能查看。
订单详情不展示库存流水号、入库状态、入库时间、来源单号等库存作业字段，避免把订单详情做成库存处理详情。

6. 库存台账

用于查看“现在仓库里还剩多少”。库存台账是当前库存结果，不强调入库 / 出库发生时间。

字段展示规则
农产品名称卡片主标题。
仓库名称辅助信息行，左侧仓库图标。
库存类型标签展示：收粮 / 存粮。
结存重量灰底信息块，字段名与数值左侧紧凑排列。
库存台账不提供汇总展示，不提供详情下钻。若要看变动时间和来源，进入库存明细。

7. 库存明细

用于查看库存变动流水，按发生时间倒序展示。右上角表达库存方向，左下角表达库存来源。

元素展示规则
出入库状态右上角标签：入库为主题绿，出库为低饱和红。
库存类型左下角标签：收粮为中性灰，存粮为浅蓝。
订单编号辅助信息行，左侧编号图标。
变动重量 / 结存量灰底信息块，字段名固定为变动重量、结存量。
变动时间完整时间，展示到秒。
详情非汇总卡片点击后从底部弹出详情。

汇总规则

汇总维度汇总后展示汇总后不展示
库存类型 + 农产品 + 仓库 + 入库 / 出库农产品、仓库、出入库状态、变动重量合计、合并笔数、库存类型标签。订单编号、结存量、流水号、变动时间、操作备注。

8. 农户存粮

用于查看农户寄存在收购商仓库内、尚未出售给收购商的存粮记录。

状态字段交互
未汇总农户姓名、电话、订单编号、存粮种类、存粮重量、存粮时长、存粮时间。点击卡片打开详情弹层。
汇总农户姓名、电话、存粮种类、存粮重量合计。点击明细进入该汇总分组下的原始明细集合。
农户存粮是寄存粮口径，不等同于收购商已买断库存。汇总态不展示单条订单编号、单条存粮时间、存粮时长和仓库单值。

数据来源移动端使用方式后端要求
仓库列表PC 仓库接口开单下拉与库存筛选，仅展示启用仓库。返回启用状态、仓库 ID、仓库名称。
订单仓库订单接口订单详情展示仓库名称快照。保存 warehouseId、warehouseName。
待确认入库库存 / 订单联动接口库存概览展示待确认入库重量。明确状态名与统计口径。
库存台账PC 库存台账接口只读展示当前结存。按权限过滤仓库范围。
库存明细PC 库存流水接口只读展示入库 / 出库流水。支持搜索、筛选、汇总。
农户存粮PC 农户存粮接口只读展示未汇总、汇总和分组明细。支持汇总和分组明细查询。

11. 原型对齐与方案补充说明

分类内容当前处理
原型已覆盖我的 > 库存管理入口、库存概览、库存台账 / 库存明细 / 农户存粮三 Tab、最近 5 条预览、全部列表、筛选、汇总、详情弹层。作为本期移动端展示基线。
原型已覆盖开单页入库仓库、订单详情按条件展示入库仓库、不开库存处理模块。作为开单与订单详情联动基线。
PRD 补充仓库禁用后历史订单仍展示仓库名称快照。需要后端保存订单仓库快照。
PRD 补充开单选择仓库后形成预入库 / 待确认入库。需要后端确认状态名、生成时机和统计口径。
PRD 补充权限过滤、直链无权限、接口失败、无数据、筛选无结果等异常态。需要前后端联动补齐。
PRD 补充趋势指标较昨日的统计时点。需要后端或数据口径确认。
原型主要表达页面结构、字段展示和交互路径；PRD 额外说明后端状态、权限过滤、历史快照、异常空态和统计口径。

12. 异常边界与待确认事项

异常场景处理规则
仓库接口失败开单页仓库字段展示加载失败提示，可重试；不影响不选仓库继续开单。
无启用仓库下拉展示空态，提示暂无可用仓库；允许不选择仓库并继续开单。
用户无库存权限不展示入口；直链进入展示无权限空态。
库存接口失败页面展示加载失败和重试入口，不展示错误数据。
无数据 / 筛选无结果展示空态，不展示新增按钮。
评审关注点需要确认的问题
预入库状态状态名、生成时机、是否进入库存概览待确认入库。
仓库接口是否可返回启用状态、权限过滤后的仓库列表。
订单接口是否保存仓库快照，历史禁用仓库是否仍可展示名称。
库存明细接口是否支持按库存类型、农产品、仓库、入库 / 出库汇总。
农户存粮接口是否支持未汇总、汇总、分组明细三种查询。
趋势指标较昨日的统计时点和计算方式。

13. 验收标准

  • 未启用入库配置时，开单页不展示入库仓库字段。
  • 启用入库配置时，开单页展示入库仓库，下拉只展示启用仓库。
  • 入库仓库默认可为空，不影响挂单、待开价、待付款、付款成功等订单流程。
  • 未选择仓库的订单，详情页不展示入库仓库。
  • 选择仓库的订单，仅在订单信息中展示入库仓库，不展示库存作业字段。
  • 我的页仅超管和有库存查询权限账号展示库存管理入口。
  • 库存管理首页展示库存概览和三 Tab 最近 5 条预览。
  • 库存管理首页不展示搜索、筛选、汇总控件。
  • 库存概览四个指标展示较昨日趋势，升绿降红。
  • 库存台账列表支持搜索和筛选，不支持汇总和详情下钻。
  • 库存明细列表支持搜索、筛选、汇总统计和非汇总详情弹层。
  • 库存明细字段名称统一为变动重量、结存量，时间展示到秒。
  • 农户存粮列表支持搜索、筛选、汇总展示、非汇总详情弹层和汇总分组明细。
  • 库存管理页不出现新增、编辑、删除、出库、调拨、盘点、存转销等操作按钮。
  • 无权限、无数据、接口失败、筛选无结果均有明确空态或错误提示。
