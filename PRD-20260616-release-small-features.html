2026-06-16 发版小功能 PRD

本次小版本主要围绕首页信息价值、首页视觉层级、关怀版大字体能力、协议内容和农户建档字段做轻量优化。目标是在不调整核心业务流程的前提下，让首页更有经营参考价值，减少无效信息和用户理解成本。

1. 功能范围

首页文案精简首页

首页下掉“农产品交易平台”展示，减少重复定位文案。

收购统计增强首页统计

收购商首页统计卡片增加本月累计收购、本年累计收购金额与笔数。

品类明细口径优化经营看板

品类明细去掉“今日”页签，保留本月、本年口径。

待办事项卡片优化待办

优化待办卡片 UI 样式，提升可读性和点击引导。

大字体关怀版用户体验

丰收云切换为大字体关怀版，提升中老年用户使用体验。

农户地址字段农户建档

农户建档页面“联系地址”统一改为“农户地址”，且农户地址调整为非必填。

身份证号正则校验农户建档

农户建档页身份证号字段补充格式校验，格式不正确时拦截提交并提示修改。

用户服务及隐私协议内容替换协议合规

替换用户服务协议及隐私协议展示内容，确保登录页、关于页等入口查看到最新协议。

2. 首页下掉“农产品交易平台”

业务目标

  • 首页减少口号式、副标题式文案，避免和“丰收云”品牌名称重复表达。
  • 首页首屏优先展示身份、经营数据、待办事项等直接有用的信息。
位置调整前调整后说明
首页顶部品牌区域展示“农产品交易平台”不展示该文案保留丰收云品牌和必要身份信息即可。
REQ-001 首页不再展示“农产品交易平台”文案。

3. 收购商统计卡片增加本月、本年累计

背景

当前首页主要强调今日收购金额，但实际收购商不一定每天都有交易，今日数据经常为 0，导致首页经营价值感不足。本次在保留今日口径的基础上补充本月、本年累计，让用户打开首页即可看到更稳定的经营数据。

字段展示内容规则
今日收购总额金额、笔数保持现有主展示，不改业务口径。
本月累计收购金额、笔数展示当月累计收购金额和订单笔数。
本年累计收购金额、笔数展示当年累计收购金额和订单笔数。
REQ-002 收购商首页统计卡片应展示本月累计收购金额及笔数。
REQ-003 收购商首页统计卡片应展示本年累计收购金额及笔数。
REQ-004 今日金额为 0 时，本月、本年累计仍正常展示。

4. 品类明细去掉今日标签

业务目标

  • 去掉数据价值较低的“今日”页签，避免默认展示大量 0 数据。
  • 品类明细保留更适合经营分析的本月、本年口径。
位置调整默认口径说明
首页品类明细卡片移除“今日”页签默认本月保留本月、本年切换；若后续只保留一个口径，优先保留本月。
REQ-005 品类明细不再展示“今日”页签。
REQ-006 品类明细至少保留本月口径，建议同时保留本年口径。

5. 待办事项卡片 UI 样式优化

业务目标

  • 待办事项需要让收购商快速判断当前需要处理什么。
  • 优化卡片视觉层级，不调整待办统计逻辑和跳转规则。
调整项要求
视觉层级待办标题、数量、辅助说明层级清晰，统一左对齐，数量突出但不压迫页面。
卡片点击保留原跳转能力，点击对应卡片进入相关处理列表。
空状态数量为 0 时样式仍完整，不出现布局塌陷。
移动端适配小屏幕下文字不重叠、不截断关键数字。
REQ-007 待办事项卡片 UI 优化不得改变原有待办数量统计与跳转逻辑。

6. 丰收云切换为大字体关怀版

业务目标

  • 丰收云切换为大字体关怀版，提升中老年用户阅读和点击体验。
  • 优先覆盖高频页面和关键流程。，尽可能覆盖全流程页面
能力规则说明
字体大小关怀版整体字号更大重点保证首页、开单、订单、我的等高频页面可读。
布局适配文字变大后不重叠、不遮挡、不撑破按钮必要时调整行高、间距和容器高度。
REQ-008 切换到大字体关怀版。
REQ-010 关怀版不得导致关键按钮、金额、订单状态文字重叠或被遮挡。

7. 农户建档页面“联系地址”改为“农户地址”且非必填

业务目标

当前字段名称“联系地址”容易理解为联系方式的一部分，而实际用于记录农户所在地或经营地址。本次统一改为“农户地址”，并将该字段调整为非必填，避免因地址暂时无法确认而阻断建档。

页面调整前调整后说明
农户建档页联系地址农户地址调整字段展示文案。
农户建档页农户地址必填农户地址非必填未填写农户地址时允许提交建档；已填写时正常保存。
REQ-011 农户建档页面字段文案由“联系地址”改为“农户地址”。
REQ-012 农户建档页面“农户地址”字段改为非必填，未填写时不拦截提交。

8. 农户建档身份证号正则校验

业务目标

农户建档页身份证号需要补充前端格式校验，减少无效证件号进入后续建档、结算或实名相关链路。

字段校验规则错误处理
身份证号支持 18 位居民身份证号码格式校验，末位支持数字或 X / x。格式不正确时拦截提交，并在字段附近提示“请输入正确的身份证号”。
REQ-013 农户建档页身份证号字段补充正则校验。
REQ-014 身份证号为空时仍按原业务必填规则处理；已填写但格式不正确时必须提示并阻断提交。

9. 用户服务及隐私协议内容替换

业务目标

替换用户服务协议和隐私协议页面内容，保证用户在登录页、关于页等入口查看到最新协议文本。

入口调整内容验收要求
登录页协议入口用户服务协议、隐私协议内容替换为最新版本。点击后展示最新协议内容，标题和正文不出现旧协议残留。
关于页协议入口用户服务协议、隐私协议内容保持一致。与登录页打开的协议内容一致。
REQ-015 替换用户服务协议内容。
REQ-016 替换隐私协议内容。
REQ-017 所有协议入口跳转到同一份最新协议内容。

10. 业务规则汇总

编号规则说明
BR-001首页不展示“农产品交易平台”。仅删除展示文案，不影响业务入口。
BR-002首页统计卡保留今日主口径，并补充本月、本年累计。不引入自定义日期筛选。
BR-003品类明细去掉今日页签。默认本月，保留本年切换。
BR-004待办卡片只做 UI 优化。不改变数量、状态和跳转逻辑。
BR-006“联系地址”改“农户地址”。字段展示文案统一调整。
BR-007农户地址改为非必填。未填写地址时允许建档提交；地图选点和手动填写能力不受影响。
BR-008身份证号补充格式校验。格式错误时拦截提交，避免无效身份证号进入后续流程。
BR-009用户服务协议和隐私协议内容统一替换。登录页与关于页等入口展示内容保持一致。

11. 验收标准

  • 首页不再出现“农产品交易平台”文案。
  • 收购商首页统计卡片能看到本月累计收购金额/笔数、本年累计收购金额/笔数。
  • 品类明细卡片不再展示“今日”页签，默认展示本月，支持本年切换。
  • 待办事项卡片视觉层级优化后，数量为 0 或较大数字时均不破版。
  • 切换大字体关怀版后，高频页面文字变大且主要按钮、金额、状态不重叠。
  • 农户建档页面字段文案展示为“农户地址”。
  • 农户建档页面农户地址为空时可以正常提交。
  • 农户建档页面身份证号格式错误时拦截提交，并展示明确错误提示。
  • 登录页、关于页进入用户服务协议和隐私协议时，展示最新替换后的协议内容。
