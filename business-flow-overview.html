丰收云从入驻、出码、扫码开单到付款成功的业务全景

本页按当前最新上线功能梳理：收购商通过企微进件，资料填写正确后直接免审开户；登录后再协助农户建档，农户出示二维码，收购商扫码开单并按实际填写情况在挂单、待开价、待付款之间流转，随后发起单笔付款，订单最终进入支付成功或已取消。

收购商 农户 扫码开单 单笔付款 订单状态

1. 当前功能边界

先把范围钉牢：本页只描述当前可用主流程，不把后续规划能力混进业务图。

包含：企微进件免审开户、手机号验证码登录、收购商建档农户、农户出示二维码、收购商扫码开单、挂单 / 待开价 / 待付款流转、单笔付款、订单状态变化。
包含：农户查看收款记录；收购商查看订单管理、付款中、已结束记录。
不包含：开单配置、一口价、在品类设置中配置开单规则。
不包含：合并付款、批量付款、合并付款状态流和多笔批量支付失败重试。

2. 完整操作业务流程图

从收购商准入开始，到农户建档、扫码开单、在挂单 / 待开价 / 待付款之间流转，再到发起付款和完成支付；信息完整时也可以直接立即付款，跳过暂存等待。

flowchart TD
  A[收购商企微进件] --> B[资料填写正确]
  B --> C[免审开户成功]
  C --> D[生成收购商账号]
  D --> E[手机号加验证码登录丰收云小程序]
  E --> F[收购商邀请农户建档]
  F --> G[农户扫码完成建档]
  G --> H[农户出示丰收码]
  H --> I[收购商开单]
  I --> J{开单方式}
  J -- 正常开单 --> K[手动选择已建档合作农户]
  J -- 扫码开单 --> L[扫描农户二维码]
  K --> M[进入开单页]
  L --> M
  M --> N[录入品类 毛重 皮重 单价等]
  N --> O{开单操作}
  O -- 暂存 --> P{暂存归属}
  O -- 立即付款且信息完整 --> Q[付款中]
  P -- 无皮重 --> R[挂单]
  P -- 有皮重无单价 --> S[待开价]
  P -- 信息完整 --> T[待付款]
  R -- 补充皮重 --> S
  R -- 直接补齐信息 --> T
  S -- 补齐单价 --> T
  T -- 发起付款 --> Q
  Q --> U{支付结果}
  U -- 支付成功 --> V[已结束-支付成功]
  U -- 支付失败继续支付 --> Q
  U -- 取消订单 --> W[已结束-已取消]
  V --> X[农户查看已收款]

4. 订单状态流程图

当前单笔开单支付链路中，待处理承接挂单、待开价、待付款；发起付款后进入付款中，直到支付成功或已取消归档到已结束。

flowchart TD
  A{开单操作}
  A -- 暂存且无皮重 --> B[挂单]
  A -- 暂存且有皮重无单价 --> C[待开价]
  A -- 暂存且信息完整 --> D[待付款]
  A -- 立即付款且信息完整 --> E[付款中]
  B -- 补充皮重 --> C
  B -- 直接补齐信息 --> D
  C -- 补齐单价 --> D
  D -- 发起付款 --> E
  E -- 支付失败继续支付 --> E
  E -- 支付成功 --> F[已结束-支付成功]
  E -- 取消订单 --> G[已结束-已取消]
  B -- 付款前取消 --> G
  C -- 付款前取消 --> G
  D -- 付款前取消 --> G
页面分组 承接状态 业务含义
收购商待处理 挂单、待开价、待付款 订单还没有完成付款。选择暂存时，无皮重进入挂单；有皮重无单价进入待开价；信息完整进入待付款。挂单可转待开价或待付款，待开价可转待付款。
付款中 付款中、支付中、处理中 已发起付款，支付失败时仍停留在该状态，可继续支付或取消。
收购商已结束 支付成功、已取消 订单最终归档，支付成功或取消后不再回到主开单流。
农户待收款 挂单、待开价、待付款、付款中 农户可全程看到收购商为自己开的单以及当前状态。
农户已收款 / 已取消 支付成功、已取消 支付成功进入已收款，取消订单进入已取消。

5. 扫码开单时序图

把现场扫码的参与方拆开看：农户只负责出码和查记录，收购商完成开单与付款，系统负责识别身份、生成订单和更新状态。

sequenceDiagram
  participant Farmer as 农户
  participant Buyer as 收购商
  participant Mini as 丰收云小程序
  participant Order as 订单服务
  participant Pay as 支付通道

  Buyer->>Mini: 手机号加验证码登录
  Buyer->>Mini: 为农户建档
  Mini-->>Farmer: 生成农户档案和收款二维码
  Farmer->>Mini: 登录并展示二维码
  Buyer->>Mini: 扫描农户二维码
  Mini-->>Buyer: 打开开单页并带入农户信息
  Buyer->>Mini: 录入品类 毛重、皮重、单价等
  Mini->>Order: 创建待处理订单
  Buyer->>Mini: 暂存为挂单/待开价/待付款
  Note over Buyer,Order: 挂单可补单价转待开价，也可直接补齐转待付款
  Note over Buyer,Order: 待开价可补齐单价转待付款
  Buyer->>Mini: 确认单笔付款
  Mini->>Pay: 发起支付
  Pay-->>Mini: 返回付款中
  Mini->>Order: 更新订单为付款中
  Pay-->>Mini: 支付成功或支付失败
  Mini->>Order: 成功则更新已结束-支付成功
  Pay-->>Mini: 失败则保留付款中
  Farmer->>Mini: 支付成功后查看收款记录

8. 当前功能地图

收购商小程序

首页待办、合作农户、扫码开单、订单管理、付款信息维护、个人中心。

农户小程序

登录、展示收款二维码、查看收款记录、维护个人和结算信息。

模块 当前定位 关键页面
收购商首页 看待办、合作农户和经营信息，不作为首页开单入口。 buyer-home.html
开单 由底部导航或扫码进入，完成品类、重量、单价、收款人等信息录入。 buyer-billing.html
订单管理 按待处理、付款中、已结束管理订单状态。 buyer-order-manage.html
农户收款 出示二维码并查看收款记录。 farmer-collection.html、farmer-collection-records.html
