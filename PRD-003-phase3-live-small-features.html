1. 功能范围

本次小迭代不是重做主链路，而是在现有三期开单、支付、订单体系上补齐运营和现场开单效率能力。

团队管理收购商超管

维护团队成员，支持查看团队列表、新增业务员和删除业务员。

付款/结算信息修改收购商 / 农户

收购商维护付款银行信息，农户维护收款结算信息。

扫码开单微信 / 邮储银行 App

扫农户二维码直达开单页，完成暂存或直接付款。

2. 团队管理

业务目标

  • 收购商超管可以维护团队成员，便于收购站管理内部业务员。
  • 团队管理入口仅面向超管展示，业务员不应具备团队管理能力。
  • 团队管理只聚焦团队成员维护，不包含其他授权配置。
  • 当前阶段一个手机号只能加入一个收购商团队，暂不支持同一手机号同时加入多个团队。

功能逻辑

模块规则说明
入口展示收购商“我的”页展示团队管理入口，且仅超管可见。业务员身份进入时不展示该入口。
成员列表展示团队成员基础信息、角色、状态等信息。角色仅保留超管和业务员。
添加业务员超管可通过手机号添加业务员。同一手机号已在当前团队或其他收购商团队中时，均不允许再次添加。
删除业务员超管可删除团队中的业务员。删除后该业务员不再出现在团队列表中；超管账号不支持被普通删除。
提醒：同一手机号跨多个收购商团队复用是后续能力，当前版本需要明确拦截。

3. 付款/结算信息修改

业务目标

  • 付款或收款失败常由姓名、手机号、身份证号、银行卡号等要素填写错误导致，需要提供纠错入口。
  • 收购商端入口命名为“付款信息修改”，页面内展示“付款银行信息”。
  • 农户端入口仍命名为“结算信息修改”，页面内展示结算信息。
  • 当前按“一人一张银行卡”设计，不从多账户列表选择。

收购商个人户付款银行信息规则

字段默认展示修改校验修改方式
姓名脱敏当前手机号验证码校验通过后展示未脱敏原值，再输入新值确认。
手机号脱敏原手机号验证码 + 新手机号验证码一步完成双验证码校验后修改。
身份证号脱敏当前手机号验证码校验通过后展示未脱敏原值，再输入新值确认。
银行卡号仅展示脱敏卡号当前手机号验证码校验通过后展示原银行卡号，输入新卡号并重新选择开户行、开户支行。
开户行 / 支行明文展示不单独修改仅在修改银行卡号时同步重选。

农户个人户结算信息规则

字段默认展示修改校验修改方式
姓名脱敏不走手机号验证码前置步骤点击修改后直接展示未脱敏原值和新值输入页。
预留手机号脱敏展示不支持修改仅展示，不提供修改按钮。
身份证号脱敏不走手机号验证码前置步骤点击修改后直接展示未脱敏原值和新值输入页。
银行卡号仅展示脱敏卡号不走手机号验证码前置步骤点击修改后直接展示原银行卡号，输入新卡号并重新选择开户行、开户支行。

对公户修改规则

  • 展示对公账户名称、手机号、银行账号、开户行、开户支行。
  • 开户行、开户支行不展示“不可修改”文案，也不提供独立修改按钮。
  • 收购商端对公户修改仍需当前手机号验证码；农户端修改直接进入编辑页。
原型短信验证码固定为 123456；真实系统需接短信验证码、银行四要素校验和操作日志。

4. 扫码开单

业务目标

业务员或收购商超管在现场收粮时，可直接使用微信或邮储银行 App 扫描农户提供的二维码，进入开单页完成开单，减少搜索农户、切换入口的操作成本。

1农户出示二维码
2微信 / 邮储银行 App 扫码
3首次进入先登录，后续免登
4直达开单页，暂存或直接付款

功能逻辑

场景规则结果
微信扫码首次进入识别登录态；无登录态时先进入登录流程。登录成功后自动进入开单页。
微信扫码再次进入已有有效登录态时免登。直接进入开单页。
邮储银行 App 首次扫码同样需要完成首次登录。登录成功后进入开单页。
邮储银行 App 再次扫码已有有效登录态时免登。直接进入开单页。
扫码后页面限制扫码开单只进入开单页，不进入其他页面。用户只能暂存或直接付款。
农户信息带入二维码解析出农户身份和默认收款账户。开单页收款人信息自动回填。

边界规则

  • 二维码无效、过期或解析失败时，应提示“二维码无效，请农户重新出示”。
  • 农户收款二维码应代表有效农户身份和默认收款账户，扫码进入时默认可带入收款人信息。
  • 未加入团队或未开通账号的手机号，在首次登录阶段无法完成登录，不进入开单页。
  • 扫码开单完成暂存后，订单进入订单管理的挂单 / 待处理链路；直接付款则进入确认付款和支付链路。

5. 订单管理关联验收点

  • 订单管理一级页签为待处理、付款中、已结束。
  • 待处理下包含挂单、待开价、待付款；挂单、待开价、待付款均可编辑信息。
  • 付款中统一承接 pending / paying / processing 等支付中间态，不再单独展示付款失败状态。
  • 付款中支持多选、全选和批量付款，适用于批量付款部分失败后再次发起付款。
  • 已结束中的取消订单统一展示为已取消，不区分付款前取消或付款中取消。

6. 验收口径

功能必须满足不在本次范围
团队管理超管可见入口并维护成员；支持新增和删除业务员；手机号不能跨团队重复添加。复杂组织架构、审批流、同一手机号加入多个团队。
付款/结算信息修改收购商按付款银行信息校验修改；农户手机号只展示，姓名、身份证号、银行卡号直接进入修改页。真实短信、真实银行四要素接口。
扫码开单微信和邮储银行 App 扫码均支持首次登录、后续免登、直达开单页。跳转到非开单页面、扫码后进入订单列表。
订单管理付款中可批量付款，按钮文案统一为批量付款。真实批量支付失败明细拆分。
